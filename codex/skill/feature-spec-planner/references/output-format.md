# output format

## plan.md

Use sections:
- Summary
- Entities
- States
- Transitions
- Invariants
- Forbidden states
- Assumptions
- Non-goals

## impl.md

Use sections:
- Required behavior
- Constraints
- Persistence expectations
- Do not do
- Suggested implementation order

## tests.md

Use sections:
- Happy paths
- Invalid transitions
- Invariant checks
- Failure cases
- Retry/idempotency
- Property-based ideas
